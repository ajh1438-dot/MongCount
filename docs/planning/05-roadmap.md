# 05. 로드맵 (Roadmap)

> **프로젝트**: MongCount (쉼일지)
> **문서 버전**: v1.0
> **최종 갱신**: 2026-04-16
> **근거**: Socrates Phase 2.5 일정 계획
> **상태**: Locked (로드맵 확정)

---

## 1. Phase A: 선행 콘텐츠 (3개월, 병렬 진행)

### 목표
- **Audience 형성**: X/Threads에 "쉼" 테마 콘텐츠 500명+ waitlist
- **마케팅 기반 구축**: 과학, 사례, 팁 중심 스토리텔링
- **인플루언서 관계 구축**: 유튜버/블로거 첫 컨택

### Month 1: 콘텐츠 기초 & 계정 운영

**주 1-2: 과학 이해 & 콘텐츠 개발**
- [ ] 논문 리딩: DMN(Default Mode Network) 관련 연구 5개
- [ ] 한국 사례 조사: 멍때리기 대회, 뇌과학 커뮤니티
- [ ] X 계정 개설 + 프로필 구성 (바이오: "쉼의 과학과 문화")

**주 3-4: 콘텐츠 제작 & 첫 배포**
- [ ] 스레드 1-5 작성: "쉼이 왜 필요한가" 시리즈 (각 5-7 트윗)
- [ ] 각 스레드당 이미지 1개 (뇌 활동도, 통계 등)
- [ ] 배포: 주 2회 (월/목)
- 목표 리치: 스레드당 2,000 임프레션

**산출물**:
- X 계정 팔로워 50명
- 스레드 5개 (좋아요 합계 200+)

---

### Month 2: 팔로워 성장 & Waitlist 신호

**주 5-6: 콘텐츠 가속 & 상호작용**
- [ ] 스레드 6-10 작성: "과학과 문화" 교차 섹션
  - "왜 한국인은 '멍때리기'라고 부를까"
  - "유튜브 쇼츠 vs 쉼: 뇌활동 비교"
  - "창의성은 휴식에서 나온다"
  - "스트레스를 받은 직장인의 쉼 습관"
  - "쉼과 생산성: 하버드 비즈니스 리뷰"
- [ ] 댓글 참여: 팔로워들의 "쉼 경험담" 엔게이징
- [ ] 리트윗 & 답글: 관련 계정 상호작용

**주 7-8: Waitlist 페이지 론칭**
- [ ] 랜딩 페이지 (Next.js + 간단):
  - 헤더: "쉬어야 일이 잘 풀려"
  - 소개: MongCount 비전 (200자)
  - 이미지: 쉬는 사람 (헤더/배경)
  - [메이신 입력] 필드
  - 뉴스레터 구독 (Mailchimp/Substack)
- [ ] X 스레드에서 랜딩 링크 공유

**목표**:
- X 팔로워 200-300명
- 스레드 월 리치 10K+
- Waitlist 가입 50명

**산출물**:
- 스레드 10개
- 랜딩 페이지
- 뉴스레터 구독자 50명

---

### Month 3: Waitlist 확대 & 인플루언서 컨택

**주 9-10: 콘텐츠 다양화 & 교육**
- [ ] 스레드 11-14 작성: "쉼 실전 가이드" 시리즈
  - "5분 쉬는 올바른 방법"
  - "쉼의 4가지 유형"
  - "직장에서 쉬기 (거부감 극복)"
  - "쉼 기록으로 자기 이해하기"
- [ ] 비디오 쇼츠: 30초 "쉼 팁" (제작 또는 큐레이션)
- [ ] 뉴스레터: 주 1회 "쉼 과학 팟캐스트 요약"

**주 11: 인플루언서 리스트 & 초대안**
- [ ] 생산성/웰니스 유튜버 10명 조사
  - 채널: 심리학, 생산성, 명상, 개인발전
  - 구독자: 10K-100K (매니지 가능한 규모)
- [ ] DM/이메일 초대안 작성
  - "쉬어야 일이 잘 풀려" 테마 강조
  - Wander120과의 차별화 지점 언급
  - 사전 액세스 제안
- [ ] 3명 이상에게 아웃리치 시작

**주 12: 콘텐츠 마무리 & Waitlist 정산**
- [ ] 스레드 15-16: "쉼일지 곧 출시" 티저 시리즈
- [ ] Waitlist 최종 카운트 (목표 500명)
- [ ] 답장 온 인플루언서 리스트 정리

**목표**:
- X 팔로워 400-500명
- Waitlist 500명 이상
- 인플루언서 피드백 3명 이상

**산출물**:
- 스레드 16개
- 비디오 쇼츠 4개
- 뉴스레터 구독자 200명
- 인플루언서 컨택 내역

## 2. Phase B: MVP 빌드 (2.5개월, 10주)

### Week 1-2: 프로젝트 기초 & Supabase 스키마

**주 1 목표**: 로컬 개발 환경 구성, 데이터 모델 확정

- [ ] 프로젝트 구조
  - Next.js 15 + TypeScript 초기화
  - Tailwind CSS + shadcn/ui 설정
  - Zustand 스토어 기초 구조
  - ESLint + Prettier 규칙

- [ ] Supabase 연동
  - PostgreSQL 테이블 생성 (users, rest_sessions, idea_records, notification_subscriptions, subscriptions)
  - Row-level Security (RLS) 정책 설정
  - Google OAuth 기본 설정

- [ ] 로컬 개발 데이터베이스
  - Supabase CLI: `supabase local start`
  - 테스트 데이터 시딩

**산출물**:
- GitHub 리포지토리 (main 브랜치)
- 로컬 개발 환경 문서
- TypeScript 스키마 정의

---

### Week 2-4: F1 강제 쉼 알림 구현

**주 2-3 목표**: Web Push 메커니즘 구성

- [ ] Serwist 설정
  - Service Worker 초기화
  - Manifest.json 작성
  - 정적 자산 pre-cache

- [ ] Web Push API
  - VAPID 키 생성 (openssl 또는 web-push npm)
  - 구독 요청 UI (알림 권한)
  - 구독 정보 Supabase 저장

- [ ] iOS PWA 메타 태그
  - apple-mobile-web-app-capable
  - apple-touch-icon 준비

**주 4 목표**: 알림 전달 & 로컬 폴백

- [ ] 백엔드: Supabase Edge Function
  - 시간대별 알림 스케줄 (06:30, 08:30, ..., 22:00)
  - 사용자별 timezone 변환
  - FCM 또는 Web Push 라이브러리 호출

- [ ] 프론트엔드: 로컬 폴백
  - setInterval 기반 시간 체크
  - iOS PWA 미지원 환경용 Notification API

- [ ] 마이크로카피 시간대별 정의

**산출물**:
- Service Worker 코드
- Web Push 구독 플로우
- Supabase Edge Function (알림 발송)
- 테스트 기록: 알림 수신 확인

---

### Week 5-6: F2 쉼 세션 타이머 + F3 기록 UI

**주 5 목표**: 타이머 & 세션 화면 (S2-S3)

- [ ] S2 타이머 화면
  - 거대한 MM:SS 표시 (requestAnimationFrame)
  - 시간 선택: 3/5/10분 선택기
  - 중단 버튼 구현
  - 다크모드 (22시 자동 전환)

- [ ] 세션 상태 관리
  - Service Worker에서 백그라운드 타이머
  - 앱 크래시 복구 로직
  - 오프라인 모드 (IndexedDB 저장)

- [ ] S3 집중도 + 한줄 필드
  - ☁︎(1) → ☀︎(5) 슬라이더 컴포넌트
  - 텍스트 입력 필드 (자동 저장, 500ms debounce)
  - 저장/건너뛰기 버튼

**주 6 목표**: 데이터 저장 & 동기화

- [ ] IndexedDB 저장
  - Dexie.js 스키마 정의
  - 세션 + 기록 동시 저장

- [ ] Supabase 동기화
  - 온라인 시 자동 POST to rest_sessions, idea_records
  - 충돌 해결: timestamp 기준 or 사용자 선택

**산출물**:
- S2-S3 UI 컴포넌트
- Service Worker 타이머 로직
- IndexedDB 저장 & 쿼리
- 통합 테스트 (타이머 정확도 검증)
