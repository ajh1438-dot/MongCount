# 01. 제품 요구사항 정의 (PRD)

> **프로젝트**: MongCount (쉼일지)
> **문서 버전**: v1.0
> **최종 갱신**: 2026-04-16
> **근거**: Socrates Phase 0.5~2.5 컨설팅 결과
> **상태**: Locked (기획 확정)

---

## 1. 제품 개요

### 이름 & 태그라인
- **공식명**: MongCount
- **국문 부제**: 쉼일지
- **태그라인**: "쉬어야 일이 잘 풀려"
- **포지셔닝**: "쉼의 습관을 가르치는 유일한 코치" (슬로건: Duolingo for Rest)

### 1문단 요약
MongCount는 바쁜 지식노동자가 **하루 6번 정해진 시간에 5분씩 쉼**을 강제하고, 쉼의 질을 기록하며, **머리가 맑아지는 경험을 매번 증명**함으로써 "더 잘 쉬는 사람"으로 성장시키는 웹기반 휴식 코칭 앱입니다. 유튜브 쇼츠 같은 도파민 트랩과 달리, **자유로운 멍때림(DMN 진입)을 기록하고 아이디어 부산물을 포착**해 창의성도 함께 증진합니다.

## 2. Problem Statement (JTBD)

### 사용자의 진정한 일:
> **"나는 머리가 복잡하고 바쁠 때, 한발 떨어져 현실을 명료하게 보고 더 좋은 성과를 내고 싶다."**

### 현재의 문제:
- **쉼 횟수 부족**: 생각날 때만 쉼 → 하루 1-2회 비의도적 쉼만 가능
- **쉼의 질 미흡**: 유튜브 쇼츠로 "쉼" 대신 → 이후 더 뿌연 머리, 집중력 저하
- **의존성 부재**: 명확한 신호/강제가 없어 쉼을 미루거나 건너뜀
- **성장 신호 부족**: 쉼이 정말 효과 있는지 알 수 없음 → 동기 저하

### 메타포:
- **쇼츠 본 후**: 뿌연 머리, 야동 본 것 같은 죄책감
- **제대로 쉰 후**: 맑은 머리, 생각 정리됨, 다음 일 시작할 의욕

### 쉼의 과학적 정의:
- **DMN(Default Mode Network) 진입**: 외부 자극 차단, 내부 성찰 모드 활성화
- **특징**: 가이드 음성 없음, 프롬프트 없음 (명상과 구별)
- **결과**: 잠재의식 아이디어 부상, 반추 완화, 감정 안정화

## 3. Target User

### Primary Persona: P1 (유일한 MVP 타겟)
- **프로필**: 9-6 책상 중심 지식노동자 (구체 직업 불특정), 회의 하루 3시간
- **경향**: 반추(rumination), 번아웃 리스크, 결과주의적 사고
- **성공 시 느낌**: 머리 맑음, 객관성 확보, 다음 일의 우선순위 명확화
- **실패 사례**: 임원 피드백 "이 정도로는 안 되지" → 과몰입 → 기분 나쁨 → 성과 저하 (주 3회 빈도)
- **현재 쉴 때 하는 것**: 유튜브 쇼츠 (자신도 "이건 쉼이 아님" 자각)
- **1년 뒤 목표**: 하루 6번 × 5분 = 30분 의도된 쉼 (습관화). 행동 변화: "잡생각 가득" → "생각 맑음"

### Persona Not in MVP
- 대학생/수험생 (시간표 구조 다름)
- 학부모 (가정과 일의 경계 모호)
- 프리랜서/자영업자 (정해진 스케줄 부재)

**제외 이유**: MVP 2.5개월 빌드 기간 내 P1 수렴 전략. v2+에서 스케줄 커스터마이징으로 확장.

## 4. Solution Overview (6-Factor Moat)

### MVP의 차별화 6가지

| # | 요소 | 상세 | 경쟁자 |
|---|------|------|--------|
| 1 | **자유 멍때림** | 가이드 음성/프롬프트 없음. 순수 침묵의 타이머만 제공 | vs Calm/Wander120 가이드 의존성 |
| 2 | **마이크로 빈도 6×5분** | 08:30·10:30·13:00·15:30·19:00·22:00 고정 시간대. 한끼는 5분씩 충분 | vs Wander120 2분(짧음), Calm 20분(김) |
| 3 | **아이디어 부산물 포착** | 쉼 직후 "한 줄 남기기" 통합. 잠재의식 창의성 기록. 경쟁자 전무 | 경쟁자 전무 |
| 4 | **한국어 + 문화 적응** | DMN 관점의 한국 비유(멍때리기 대회). 온보딩 과학 퍼스트 | vs Wander120 영어 중심 |
| 5 | **RepCount급 3초 UX** | 기록 완료까지 3초 이내. 마찰 최소화. 경쟁자 전무 | 경쟁자 전무 |
| 6 | **"더 잘 쉬는 사람" 코칭** | 주간 리포트 + AI 코멘트. 단순 트래커가 아닌 성장 도구 | vs 단순 타이머/트래커 |

**해자의 핵심**: ③(쉼-창의성 연결) + ⑤(UX) + ⑥(콘텐츠) 3개 결합. 복제 난이도 3-6개월.

## 5. MVP Scope

### Core Features (필수)
| ID | 기능 | 설명 |
|-----|------|------|
| F1 | 강제 쉼 알림 | 6회 고정 시간 푸시 + 시간대별 마이크로카피 |
| F2 | 쉼 세션 + 집중도 기록 | 5분 기본 타이머 (3/5/10분 선택). 종료 시 ☁︎(1)~☀︎(5) 슬라이더 |
| F3 | 아이디어 부산물 포착 | 집중도 기록 직후 선택 텍스트 필드. 비우고 넘어갈 수 있음 |

### Support Features (지원)
| ID | 기능 | 설명 |
|-----|------|------|
| S1 | 홈 대시보드 | "오늘 n/6 ☁︎☁︎☀︎" + "지금 3분 쉬기" 대형 버튼 + 다음 알림 시간 |
| S2 | 주간 리포트 | 요일별 참석 도트(●◐◌─) + 평균 맑기 + AI 코멘트 |
| S3 | 다크모드 | 자동 22시 전환. 22시 알림 밝기 감쇠 |
| S4 | 7일 아카이브 + 검색 | 날짜 그룹핑. 7일 초과 시 Paywall (유료) |

### Paid Features (₩2,900/월 시작, 7일 무료)
| ID | 기능 | 설명 |
|-----|------|------|
| P1 | 무제한 아카이브 | 7일 초과분 해제 |
| P2 | 음성 녹음 / 사진 첨부 | Supabase Storage 활용 |
| P3 | CSV / Notion 내보내기 | 데이터 자유 이동 |
| P4 | 멀티 디바이스 실시간 동기화 | Supabase Realtime (최대 3대) |
| P5 | 월간 리포트 + 추세 그래프 | 심화 분석, 월간 비교 |

### Deferred (v2+)
- Rumination(반추 생각 파킹) — Phase 1.5에서 MVP 제외 결정
- 프롬프트/가이드 음성 — **영구 제외** (차별화 위배)
- 게임화(뱃지/랭킹/소셜)
- 캘린더/업무앱 연동

## 6. Success Metrics (1년 뒤)

### 사용자 행동
- **일일 쉼 빈도**: 6회 알림 중 3회 이상 완료 → Soft Streak 유지
- **총 쉼 시간**: 월 150분 (주 30분) 이상 축적
- **기록율**: 세션 75% 이상에서 집중도 기록
- **아이디어 포착율**: 월 아이디어 3개 이상 (한 줄 기록)

### 제품 지표
- **DAU/MAU**: DAU 1,500 / MAU 10,000 (한국 시장 가정)
- **리텐션**: D7 50%, D30 40%, D90 25% (참고: 개인화 피트니스 앱 D30 평균 25%)
- **유료전환**: MAU × 4% = 400명 (월 ₩116만 수익)
- **Churn**: 월 5% 이하

### 정성적 성과
- **P1 피드백**: "이제 쉬면 머리가 맑아지는 게 느껴진다"
- **App Store 평점**: 4.5/5 이상
- **추천 친구**: "쉬어야 일이 잘 풀려" 톤의 입소문

## 7. Non-Goals (명시적 제외)

### 기능 관점
- ❌ 가이드 음성/명상 명령어 (DMN 자유성 훼손)
- ❌ 게임화 (뱃지/리더보드/소셜) → P1 개인 성장에만 포커스
- ❌ AI 명상/힐링 음악 (쇼츠와 동일한 도파민 함정)
- ❌ 스케줄 자동 설정/AI 추천 시간대 (고정 6회 원칙)

### 시장 관점
- ❌ B2B 기업 웰니스 패키지 (v2+ 옵션)
- ❌ 팀/소셜 기능 (개인 성장 우선)
- ❌ 캘린더/Gmail/Slack 연동 (초기 스코프 외)

### 기술 관점
- ❌ 스마트워치 앱 (PWA 기본 제약)
- ❌ 위젯/잠금화면 기능 (iOS PWA 미지원 → v2+ 옵션)
- ❌ 오프라인 전체 동기화 (로컬 우선, 온라인 시 병합)

## 8. Constraints (제약사항)

### 기술 제약
- **PWA 선택의 대가**:
  - iOS 16.4+만 푸시 지원 (구버전 앱 내 알림만)
  - iOS 홈화면 추가 필수 → 온보딩에서 강제 유도
  - 푸시 신뢰성: iOS 70%, Android 95%+
  - **Windows 미지원** (제외 패용)
  
### 리소스 제약
- **팀**: 풀스택 1명 (사이드 프로젝트 기준)
- **기간**: 2.5개월 MVP 빌드 (Phase A 3개월 선행 제외)
- **비용**: 초기 $0, MAU 10K 시 월 $15 정도 (Supabase 무료 티어)

### 비즈니스 제약
- **사이드 프로젝트**: 호응 좋으면 C(스타트업) 전환 검토
- **유료전환**: ₩2,900/월 (최소 가격, 6개월 후 ₩4,900 인상 검토)
- **Cold-start**: 개인 네트워크 0 → Audience-first GTM 필수

## 9. Known Risks

### Risk 1: Cold-start 문제
- **설명**: 초기 사용자 0에서 시작. 앱스토어 기본 추천 불가능
- **영향**: 런칭 후 3개월 내 MAU 100명 미만 가능성
- **대응**: Phase A에서 선행 콘텐츠(X 스레드) 500+ waitlist 목표

### Risk 2: iOS PWA Push 신뢰성
- **설명**: iOS 홈화면 추가 필수. 미추가 시 푸시 완전 불가능
- **영향**: 핵심 기능(F1 알림)이 동작 안 함
- **대응**: 온보딩 3단계에서 iOS 홈화면 추가 가이드 UX 필수

### Risk 3: Wander120 추격
- **설명**: 뉴질랜드 경쟁자가 6개월 내 한국어/아이디어기록 추가 가능
- **영향**: MVP 차별화 70% 소멸
- **대응**: MVP 빠른 출시(2.5개월), 즉시 PR(3명 유튜버)

### Risk 4: 6회/일 알림 피로
- **설명**: "쉠"이 강제처럼 느껴짐 → 푸시 거부
- **영향**: 리텐션 악화 (가정 D7 30% → 20%)
- **대응**: Soft Streak 3회 이상 달성 시 유지 (6/6 비요구). A/B 테스트 Phase C에서

### Risk 5: 가격 너무 낮음
- **설명**: ₩2,900은 월 0.24센트 = 가치 정당화 부담
- **영향**: 사용자가 "무료 앱" 심리로 접근 → 리텐션 낮음
- **대응**: 6개월 후 ₩4,900으로 인상 실험. 기존 사용자 grandfathering

### Risk 6: Supabase 무료 티어 한계
- **설명**: MAU 50K 넘으면 유료 요금제 이전 필수
- **영향**: 성공 시 월 인프라 비용 $60+ (사이드 부담)
- **대응**: MAU 20K 시점에 가격 인상 검토

## 10. Dependencies & Assumptions

### 기술 가정
- Supabase 서비스 지속 가능 (2년 이상)
- Web Push API 모던 브라우저 지원 (iOS Safari 16.4+, Chrome 50+)
- IndexedDB 용량 50MB 이상 (5년 아카이브 저장 가능)

### 사용자 가정
- P1은 스마트폰을 손에 쥐고 있는 시간: 하루 4시간+ (알림 가능)
- P1의 피로도: 임원 피드백 후 1주일 내 회복 필요 (반추형이므로)
- P1의 현재 쇼츠 시청: 하루 20-40분 (=놀려는 욕구 있음)

### 시장 가정
- 한국 지식노동자(9-6) 인구: ~300만 명
- MVP 타겟: 그 중 번아웃/반추 경향 20% = 60만 명
- 도달 가능(디지털 활동)?: 그 중 40% = 24만 명
- 가입 의지?: 그 중 5% = 12,000명 (MAU 10,000은 이 중 80%)
