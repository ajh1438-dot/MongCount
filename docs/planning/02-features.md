## 2. Core Feature Detail (F1-F3)

### F1. 강제 쉼 알림 (Forced Rest Notification)

**목적**: P1이 의도적으로 쉼을 미루는 행동을 강제하여 하루 6회 쉼 기회 제공

**User Story**:
> 책상에 앉아 이메일을 확인하다가, 08:30에 핸드폰에 "5분 숨 고르고 가요"라는 푸시 알림을 받는다. 
> 나는 지금 당장 쉬지 않고 싶지만, 알림이 명확하니 "지금 쉬기"를 탭한다. 
> 타이머가 시작되고, 5분 후 집중도 슬라이더(☁︎~☀︎)를 기록한다.

**Acceptance Criteria**:
- [ ] 6회 고정 시간: 08:30, 10:30, 13:00, 15:30, 19:00, 22:00 (UTC+9 기준)
- [ ] 각 시간대별 고유한 마이크로카피 제공:
  - 08:30: "5분 숨 고르고 가요"
  - 10:30: "한 번 멈춰서 정리할까요"
  - 13:00: "점심 후 몸과 마음 리셋"
  - 15:30: "오후 경항 여기까지, 깊게 쉬어봐요"
  - 19:00: "일의 끝, 쉼의 시작"
  - 22:00: "오늘 하루 천천히 돌아봐요"
- [ ] 푸시 알림 (OS 레벨) + 앱 내 알림 폴백 (오프라인/PWA 미지원 환경)
- [ ] 스누즈 기능: 15분 재알림, 슬롯당 최대 3회 (총 45분까지)
- [ ] 23시 이후 스누즈 자동 무시 (야간 피로 방지)
- [ ] 놓친 알림: 조용히 넘어감. 메시지/재시도 없음 (사용자 최종 결정 존중)

**UX 노트**:
- 푸시 알림 탭 → [지금 쉬기] / [15분 뒤] 액션 버튼
- 알림 액션 선택 → F2 쉼 세션 직행 또는 15분 후 재알림
- 다크모드: 22시 알림 이후 화면 밝기 자동 감쇠 (Blue Light 감소)

**기술 구현**:
- Web Push API + VAPID 키 (Android Chrome/Edge)
- iOS: Supabase Scheduled Functions + ServiceWorker 폴백
- 로컬 Notification fallback (FCM/APNs 미지원 환경)
- 시간대 자동 계산: 사용자 timezone 감지, UTC 변환

### F2. 쉼 세션 + 집중도 기록 (Rest Session & Clarity Recording)

**목적**: 사용자가 정해진 시간 동안 집중을 방해하지 않는 순수 쉼을 경험하고, 종료 후 자신의 "머리 맑기"를 정량화

**User Story**:
> 알림을 탭하고 쉼 세션 화면(S2)에 진입한다. 
> 거대한 "05:00" 타이머와 다크모드 배경이 보인다. 
> 나는 깊게 숨을 쉬고, 아무것도 하지 않는다. 
> 타이머가 끝나면 자동으로 집중도 화면(S3)으로 진행한다.

**Acceptance Criteria**:
- [ ] 세션 시간 선택: 3분 / 5분(기본) / 10분
- [ ] 시각: 거대한 숫자 (MM:SS 형식), 다크모드 우선 (22시 이후 자동)
- [ ] 진행 표시: 원형 progress bar 또는 선형 바 (음소거 시작 권유)
- [ ] 중단 버튼: "쉼 중단" (집중도 자동 1-2 제안)
- [ ] 세션 중 전화 수신: 자동 일시정지 → 통화 종료 시 "계속하시겠습니까?" 다이얼로그
- [ ] 세션 중 앱 전환: 백그라운드 타이머 계속 진행 (화면 꺼도 OK)
- [ ] 세션 중 화면 잠금: 타이머 계속 동작
- [ ] 타이머 완료 → 자동으로 F3(기록 화면)으로 이동

**UX 노트**:
- 세션 중 극도로 심플: 타이머 숫자 + 중단 버튼만 (음성/음악 없음)
- 띠 텍스트: "생각 떠오르면 잊지 말고" (DMN 진입 안내 역할)
- 음성 없음 (핵심 차별화): 침묵이 본 기능

**기술 구현**:
- setInterval + requestAnimationFrame (60fps 부드러운 카운트다운)
- Service Worker: 세션 중 앱 폐쇄/크래시 복구 (남은 시간 계속 계산)
- WakeLock API: 화면 켜짐 유지 (배터리 소비 고려, 토글 옵션)

### F3. 아이디어 부산물 포착 (Idea Capture)

**목적**: DMN 진입 중 뇌에서 뜨오른 창의적 아이디어/생각을 빠르게 기록하여 "쉼 = 무용지물"이 아닌 "쉼 = 성과 창출"로 reframe

**User Story**:
> 5분 쉼을 마친 후 집중도 슬라이더를 선택한다. 
> 슬라이더 아래에 텍스트 필드가 보인다: "한 줄로 남길 말씀 있으신가요?"
> 나는 "그 프로젝트 접근법을 이렇게 해볼까" 한 줄을 입력하고 [저장]을 탭한다. 
> 2초 후 다음 일상으로 복귀한다.

**Acceptance Criteria**:
- [ ] 집중도 슬라이더(F2 종료 직후) 다음에 나타남
- [ ] 단일 텍스트 필드 (기분/생각 구분 없음, "한 줄" 통합)
- [ ] 선택 사항 (비우고 넘어갈 수 있음)
- [ ] 최대 280자 (트윗 길이)
- [ ] 자동 저장: 1글자마다 localStorage debounce 500ms
- [ ] 저장/건너뛰기 버튼 (또는 자동 완료)
- [ ] 완료까지 3초 이내 (총 F2+F3 = 3초)

**UX 노트**:
- 레이블: "한 줄로 남길 말씀 있으신가요?" (부드러운 톤)
- 기분도 생각도 OK (사용자가 정함)
- 글자수 카운터 (선택)
- 오토포커스: 필드 자동 활성화

**기술 구현**:
- textarea 요소 (자동 높이 조절)
- IndexedDB 저장 + Supabase 비동기 동기화
- 오프라인 모드: localStorage debounce 후 온라인 시 일괄 동기화

## 3. Support Feature Detail (S1-S4)

### S1. 홈 대시보드 (Home Dashboard)

**목적**: 사용자가 앱 진입 시 "오늘 나는 얼마나 쉬었는가"를 한눈에 파악하고, 즉시 쉼을 시작할 수 있는 진입점

**주요 컴포넌트**:
- **상단 그리팅**: "안녕하세요, [이름]" 또는 "오늘도 좋은 하루"
- **진행도 표시**: "오늘 3/6 ☁︎☁︎☀︎" (완료 슬롯 + 아이콘)
- **대형 CTA**: "지금 3분 쉬기" 버튼 (초초색, 항상 활성)
- **다음 알림**: "다음 알림: 13:00 (30분 뒤)"
- **최근 쉼**: "마지막 쉼: 10분 전 ☀︎☀︎☀︎"
- **탭 네비게이션** (하단):
  - 홈 (활성)
  - 아카이브
  - 리포트 (주간)
  - 설정

**States**:
- 정상: 위 구성
- 첫 실행: "오늘의 첫 쉼을 시작해보세요"
- 모든 알림 완료: "오늘 쉼을 모두 완료했어요 🌟"

**기술 구현**:
- Zustand 상태: `restSessions[]`, `userPreferences`
- Real-time: Supabase Realtime (멀티디바이스 시 즉시 반영)

---

### S2. 주간 리포트 (Weekly Report)

**목적**: P1이 주간 쉼 패턴을 시각적으로 보면서 "더 잘 쉬는 사람"으로의 성장을 자각

**주요 컴포넌트**:
- **요일별 도트**: 월~일, 각 요일별 참석 상태
  - ● (완료: 6/6 또는 Soft Streak 3/6)
  - ◐ (부분: 1-2회)
  - ◌ (미참석)
  - ─ (예정)
- **평균 맑기**: "이번 주 평균 ☀︎☀︎☀︎.5" (숫자 + 아이콘)
- **최고 시간대**: "가장 맑았을 때: 목요일 15:30 ☀︎☀︎☀︎☀︎☀︎"
- **AI 코멘트** (1줄, 서버 생성):
  - "꾸준함이 빛나는 주였어요"
  - "월요일부터 멀리해도 괜찮아요" (낮은 주에는 격려)
- **월간 탭** (Paywall P5): 4주 비교, 추세 그래프

**기술 구현**:
- Supabase Function: 주 집계 쿼리 (sun-sat 자동)
- AI 코멘트: GPT-3.5 Turbo (한국어, 짧은 톤)

---

### S3. 다크모드 (Dark Mode)

**목적**: 22시 이후 사용자의 눈 피로를 줄이고, DMN 진입을 돕는 수면친화적 환경 제공

**구현 사항**:
- **자동 전환**: 22:00 ~ 08:00 자동 다크모드 (Prefers-color-scheme 우선)
- **수동 토글**: 설정에서 Light / Dark / System 선택 가능
- **색상 팔레트**:
  - 다크: 배경 #0F0F0F, 텍스트 #E8E8E8 (순수 검정 X, 눈 부담 최소)
  - 밝기 감쇠: 22시 알림 시 RGB 값 자동 감소 (Blue Light 차단)
- **타이머 화면**: 다크모드에서 가장 큰 emphasis

**기술 구현**:
- CSS 변수 + Tailwind dark: 모드
- localStorage에서 선호도 저장

### S4. 7일 아카이브 + 검색 (Archive & Search)

**목적**: P1이 최근 쉼 기록을 되돌아보고, 패턴을 발견하며, 7일 초과 저장에는 유료 전환 유도

**주요 컴포넌트**:
- **날짜 그룹핑**: "어제", "2일 전", "일주일 전" 등 상대 시간
- **각 세션 카드**:
  - 시간: "10:30"
  - 집중도: ☁︎☁︎☀︎ (아이콘만)
  - 한줄 미리보기: "프로젝트 접근법..."
- **검색 필터**: 
  - 텍스트 검색 (한줄 기록)
  - 집중도 필터 (☀︎☀︎☀︎ 이상만)
  - 시간대 필터 (아침/오후/저녁)
- **7일 경계**: "7일 이상 저장하려면 프리미엄 업그레이드 필요" (Paywall S10)

**기술 구현**:
- Dexie.js: 로컬 IndexedDB 7일 쿼리
- Supabase: 로그인 시 무제한 저장 (Free) → 아카이브 지표 업데이트

## 4. Paid Feature Detail (P1-P5)

### P1. 무제한 아카이브 (Unlimited Archive)
- 7일 초과 분 해제
- 365일 저장 가능
- 연간 리뷰 기능

### P2. 음성 녹음 / 사진 첨부 (Media Attachment)
- 한줄 기록 대신 또는 추가로 음성/사진 업로드
- Supabase Storage (GB당 비용, 초기는 50GB 포함)
- 5분 음성 녹음 가능

### P3. CSV / Notion 내보내기 (Export)
- 아카이브 전체 CSV 다운로드
- Notion API 연동 (자동 데이터베이스 생성)
- 월 1회 자동 export

### P4. 멀티 디바이스 실시간 동기화 (Multi-Device Sync)
- Supabase Realtime (WebSocket)
- 최대 3대까지 동시 동기화
- 한 디바이스에서 기록하면 다른 디바이스에서 즉시 반영

### P5. 월간 리포트 + 추세 그래프 (Monthly Report)
- 월간 집계: 총 쉼 시간, 최고 일일 빈도
- 추세 그래프: 월별 평균 맑기
- 기간별 비교: 이번 달 vs 지난달

## 5. Edge Cases (9개 명시 결정)

### E1: 세션 중 전화 수신
**결정**: 자동 일시정지 → 통화 종료 시 "계속하시겠습니까?" 다이얼로그
**근거**: 사용자 선택권 존중, 강제 중단 X

### E2: 세션 중 앱 전환
**결정**: 백그라운드 타이머 계속 진행 (화면 꺼도 OK)
**근거**: 쉼 중 유튜브 확인 같은 실수 방지, RepCount 3초 원칙 유지

### E3: 스누즈 최대 회수
**결정**: 슬롯당 최대 3회 (15분 × 3 = 45분까지)
**근거**: 무한 미루기 방지, 그 날의 쉼 완전 포기 방지

### E4: 23시 이후 알림
**결정**: 자동 무시 (야간 쉼 알림 비권장)
**근거**: 수면 방해 방지, 22시 알림이 최후

### E5: 놓친 알림
**결정**: 조용히 넘어감 (재시도/제안 없음)
**근거**: 사용자 자율성 존중, Soft Streak 3회로 유연화

### E6: 세션 중단 시 집중도
**결정**: 자동으로 1-2 제안 (사용자 확인)
**근거**: 의도하지 않은 중단과 의도한 중단 구분

### E7: 오프라인 세션 기록
**결정**: 로컬 저장 후 온라인 시 자동 동기화
**근거**: 완전 오프라인 환경도 서비스 지속

### E8: 로그인 없이 사용
**결정**: 가능 (로컬 저장만). 로그인 시 클라우드 동기화 제안
**근거**: Friction 최소화, 신뢰 구축

### E9: PWA 홈화면 미추가 (iOS)
**결정**: 온보딩 S7에서 강제 가이드. 미추가 시 앱 내 알림만 동작
**근거**: 푸시 알림 필수 → 홈화면 추가 필수

## 6. Design Taste Decisions Log

### 집중도 UI: 별점 vs 감정 아이콘
**결정**: ☁︎(1) → ☀︎(5) 감정 아이콘 선택
**이유**: 
- "뿌연/맑은" 메타포 강화
- 별점(⭐)은 "평가"의 느낌 → "상태 기록"으로 reframe

### 한줄 필드: 기분 vs 생각 분리 vs 통합
**결정**: 단일 통합 필드 ("한 줄로 남길 말씀")
**이유**:
- RepCount 3초 UX 원칙 (필드 분리 시 선택 friction +2초)
- P1의 심리: 기분/생각이 혼재 (구분 비자연스러움)
- 사용자가 필요한 것만 기록 (자유도)

### 로그인: 필수 vs 선택
**결정**: 선택 ("나중에" 버튼)
**이유**:
- 첫 사용자 friction 최소화 (즉시 쉼 경험)
- 로그인 시 클라우드 동기화 = 부가가치 (유인력 높음)

### Streak 계산: 6/6 vs Soft Streak 3/6
**결정**: Soft Streak 3/6
**이유**:
- 6회 완벽주의 → 하루 하나 놓치면 streak 깨짐 → 포기심리 유발
- 3회 이상 = 의도된 쉼 성공 (알림 피로 완화)
- A/B 테스트: Phase C에서 검증 예정

### 알림 지속성: 재시도 vs 조용히
**결정**: 조용히 넘어감 (재시도/제안 없음)
**이유**:
- Soft Streak 3회로 유연화 → 하루 놓쳤다고 강박 없음
- Push fatigue 방지 (경쟁자 추격 방지)
- Wander120과의 차별화 (non-aggressive approach)

### 기술 스택: Native App vs PWA
**결정**: PWA (Next.js + Supabase)
**이유**:
- 개발 속도 (풀스택 1명 기준)
- 배포 비용 0 (Vercel 무료)
- 웹 링크 공유 가능 (마케팅 용이)
- iOS 16.4+ 푸시 지원 (임계값)

### 아이디어 필드 위치: 별도 화면 vs 집중도 아래
**결정**: 집중도 슬라이더 바로 아래 (통합)
**이유**:
- 세션 완료 후 가장 창의적일 때 (DMN peak)
- 3초 UX 원칙 준수
- 전환율 higher (별도 화면 시 -40% 추정)
