## 1. Primary User Flows

### Flow 1: 첫 실행 (First-time Onboarding)

```
앱 실행
  ↓
[S7-1] 온보딩 스크린 1: DMN 과학
  "쉴 때 뇌의 활동이 가장 활발하대요"
  (뉴로사이언스 교육, 쉼의 가치)
  ↓
[S7-2] 온보딩 스크린 2: 알림 권한
  "하루 6번, 5분씩. 쉬는 시간 알림 드릴게요"
  [알림 허용] / [건너뛰기]
  → 허용 시: iOS 홈화면 추가 가이드 (이미지)
  ↓
[S7-3] 온보딩 스크린 3: 시작
  "언제든 '지금 3분 쉬기' 버튼 있어요"
  [시작하기]
  ↓
[S8] 로그인 스크린 (선택)
  Google OAuth / Kakao OAuth / [나중에]
  → "나중에": 로컬 유저로 즉시 홈 진입
  → 로그인 선택: 클라우드 동기화 설정
  ↓
[S1] 홈 대시보드
  "오늘 0/6 ─────"
  [지금 3분 쉬기]
  ✅ 완료
```

---

### Flow 2: 일일 사용 (Daily Use: Notification → Rest → Record)

```
[알림 트리거]
  08:30 / 10:30 / 13:00 / 15:30 / 19:00 / 22:00
  (또는 사용자 정의, MVP는 고정)

[S4] OS 푸시 알림 (또는 앱 내 알림)
  "5분 숨 고르고 가요"
  [지금 쉬기] / [15분 뒤] / [닫기]
  
┌─────────────────────────────────────────┐
│ Case A: 사용자가 [지금 쉬기] 탭         │
└─────────────────────────────────────────┘
  ↓
[S2] 쉼 세션 타이머
  시간 선택 여부:
  - 첫 실행: 기본 5분
  - 이후: 마지막 선택 기억 (3/5/10분)
  
  타이머 시작 (부드러운 애니메이션)
  "05:00" → "04:59" → ... → "00:00"
  
  극도로 심플 (타이머 숫자 + 중단 버튼만)
  띠 텍스트: "생각 떠오르면 잊지 말고"
  
  세션 중 가능한 행동:
  - 화면 꺼짐: 타이머 백그라운드 진행
  - 전화 수신: 자동 일시정지 → 통화 종료 후 "계속?" 다이얼로그
  - 앱 전환: 타이머 계속 (백그라운드)
  - [중단]: "쉼 중단" 버튼 (집중도 자동 1-2 제안)
  ↓
[S3] 집중도 + 한줄 기록
  슬라이더: ☁︎(1) → ☀︎(5)
  필드: "한 줄로 남길 말씀 있으신가요?" (선택)
  
  버튼: [저장] / [건너뛰기]
  또는 자동 저장 (500ms debounce)
  ↓
완료 (총 3초 이내)
  ↓
[S1] 홈 복귀
  진행도 업데이트: "오늘 1/6 ☀︎─────"
  
┌─────────────────────────────────────────┐
│ Case B: 사용자가 [15분 뒤] 탭           │
└─────────────────────────────────────────┘
  ↓
15분 후 재알림 (최대 3회까지, 총 45분)
  ↓
Case A로 진행하거나 [닫기]
  ↓
[S1] 홈 유지 (또는 다른 작업 계속)

┌─────────────────────────────────────────┐
│ Case C: 사용자가 [닫기] 또는 무시         │
└─────────────────────────────────────────┘
  ↓
Soft Streak 계산:
  - 하루 3회 이상 완료 → Streak 유지
  - 2회 이하 → Streak 끊김 (강박 X)
  ↓
[S1] 홈 유지
```

### Flow 3: 로그인 & 클라우드 동기화 (Logged-in Sync)

```
[S1] 홈 대시보드
  [설정] 탭 → [계정] 섹션
  
[계정 미연결 상태]
  "클라우드 동기화를 켜면 모든 기기에서 기록을 볼 수 있어요"
  [Google 로그인] / [Kakao 로그인]
  ↓
[OAuth 플로우]
  - Google: Supabase 기본 제공
  - Kakao: Supabase Edge Function (커스텀)
  ↓
[로그인 완료]
  ↓
[클라우드 동기화]
  로컬 IndexedDB → Supabase PostgreSQL
  (기존 로컬 기록 자동 업로드)
  ↓
[멀티 디바이스]
  다른 기기에서 로그인 시:
  ← Supabase 계정의 모든 기록 다운로드
  ↓
[실시간 동기화 (Paid P4)]
  한 기기에서 기록 → 다른 기기에서 3초 내 반영
  (Supabase Realtime WebSocket)
```

---

### Flow 4: 유료 전환 (Paid Conversion)

```
[시나리오 1: 7일 초과 아카이브]
  [S1] 홈 → [아카이브] 탭
  ↓
  8일 전 기록을 스크롤
  ↓
  "7일 이상 저장하려면 프리미엄이 필요해요"
  [7일 무료 체험 시작] / [나중에]
  ↓
[S10] Paywall 스크린
  5개 Paid 기능 체크리스트:
  ✓ P1 무제한 아카이브
  ✓ P2 음성 녹음 / 사진
  ✓ P3 CSV/Notion 내보내기
  ✓ P4 멀티디바이스 동기화
  ✓ P5 월간 리포트 + 그래프
  
  가격: "월 ₩2,900 / 연 ₩28,900 (저장 3개월)"
  [7일 무료 체험 시작]
  ↓
  Stripe 결제 페이지 (또는 인앱 결제 검토)
  ↓
[시나리오 2: 설정 → 업그레이드]
  [S1] → [설정] → [업그레이드]
  ↓
  S10 Paywall로 직행
  ↓
[결제 완료]
  ↓
  무제한 아카이브 해제
  (P2-P5는 자동 활성화)
  ↓
[S1] 홈 복귀
  "프리미엄 활성화 ✨" 토스트 메시지
```

## 3. State Transitions & Edge Cases

### State 1: 세션 중 전화 수신

```
[S2] 쉼 세션 진행 중
  타이머: "03:45"
  ↓
[전화 수신]
  ↓
  [자동 일시정지]
  타이머 시각: "⏸ 03:45"
  ↓
[전화 통화 중]
  앱 백그라운드
  ↓
[전화 통화 종료]
  ↓
[다이얼로그 표시]
  "쉼을 계속하시겠습니까?"
  [계속] / [중단]
  ↓
  Case A: [계속] → 타이머 재개
    "03:45" → "03:44" → ...
  Case B: [중단] → S3로 이동
    자동 집중도 1-2 제안
```

---

### State 2: 세션 중 앱 전환

```
[S2] 쉼 세션 진행
  타이머: "04:30"
  ↓
[사용자가 홈 버튼 누르거나 다른 앱 실행]
  ↓
[앱 백그라운드]
  Service Worker가 타이머 계속 계산
  ↓
[사용자가 MongCount 앱 다시 열기 (2분 후)]
  타이머: "02:30" (정확히 계속 진행됨)
  ↓
  화면 복구: "02:30" 표시
  ↓
[계속 진행 또는 중단]
```

---

### State 3: 세션 중 오프라인 + 화면 잠금

```
[S2] 쉼 세션 (Wi-Fi 켜짐)
  타이머: "05:00" 시작
  ↓
[30초 후, Wi-Fi 끊김]
  ↓
  Service Worker + Local timer 백업으로 계속 진행
  ↓
[화면 잠금 (1분 후)]
  ↓
  OS 레벨 타이머 계속 동작 (WakeLock API 미사용 시)
  ↓
[화면 다시 깨우기 (3분 후)]
  타이머: "02:00" (정확히 경과)
  ↓
  정상 복구, 계속 진행
```

---

### State 4: 크래시 복구

```
[S2] 쉼 세션 진행 (03:15)
  ↓
[앱 크래시 (OS 강제 종료 등)]
  ↓
[사용자가 앱 재실행]
  ↓
[Service Worker 복구]
  남은 시간 재계산 (크래시 시점 ~ 현재)
  ↓
[복구 다이얼로그 또는 자동 복구]
  "쉼을 계속하시겠습니까? (1분 21초 남음)"
  [계속] / [건너뛰기]
  ↓
  정상 복구 또는 S3로 이동
```

---

### State 5: 오프라인 기록

```
[S2-S3] 오프라인 환경에서 쉼 세션 + 기록
  ↓
[LocalStorage + IndexedDB 저장]
  네트워크 동기화 플래그: false
  ↓
[온라인 복구]
  ↓
[자동 동기화]
  로컬 기록 → Supabase (백그라운드)
  플래그: true
  ↓
  동기화 완료 토스트 (선택)
```

---

### State 6: 로그인 도중 앱 종료

```
[S8] 로그인 (OAuth 팝업)
  ↓
[사용자가 팝업 닫음]
  ↓
[로그인 미완료]
  ↓
[로컬 저장 유지]
  로그인 상태: false (로컬 유저 유지)
  ↓
[설정에서 재시도]
  로그인 버튼 다시 표시
```

---

### State 7: 멀티 디바이스 충돌

```
[디바이스 A] 로그인, 현재 시각 08:30
  기록 1개 저장
  ↓
[디바이스 B] 로그인, 현재 시각 08:32
  기록 2개 저장
  ↓
[Supabase Realtime]
  B의 기록이 A로 동기화 (3초)
  A의 기록이 B로 동기화 (3초)
  ↓
[최종 상태]
  A, B 모두: 기록 3개 (timestamp 기준 병합)
  충돌 해결: 최신 timestamp 우선 또는 모두 유지
```

## 4. Interaction Patterns

### Microinteractions
- **세션 종료**: 부드러운 fade-out + 자동 S3 이동 (1초)
- **슬라이더 선택**: 실시간 이모지 변화 (☁︎ → ☀︎)
- **저장 완료**: 체크마크 + 토스트 메시지 (2초 자동 소멸)
- **알림 도착**: 시스템 알림 + 앱 아이콘 배지 (숫자)

### Animation Guidelines
- **타이머**: 부드러운 카운트다운 (프레임 드롭 없음)
- **화면 전환**: 슬라이드 또는 fade (경계 명확)
- **다크모드 전환**: 자연스러운 감쇠 (즉시가 아닌 1초)

## 5. Error Handling Flows

### Network Error
```
[S1] 홈에서 데이터 로드 시도
  ↓
[네트워크 연결 없음]
  ↓
[캐시된 데이터 표시]
  "마지막 동기화: 1시간 전"
  ↓
  [재시도] 버튼 (또는 자동 백그라운드 재시도)
```

### Notification Permission Denied
```
[온보딩 S7-2]
  [알림 허용]
  ↓
[사용자가 거부]
  ↓
[S9] 설정에서 수동 활성화 유도
  "알림을 켜면 놓치지 않아요"
  [iOS 설정 열기]
```

### Storage Full (IndexedDB 50MB 한계)
```
[아카이브 저장 시도]
  ↓
[IndexedDB 용량 초과]
  ↓
  "저장소가 가득 찼어요. 오래된 기록을 삭제하거나 프리미엄으로 이동하세요"
  [파일 관리] / [업그레이드]
```
